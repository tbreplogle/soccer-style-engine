# Soccer Style Engine — Phase 3 Agent Foundation

This phase adds the first **Team Identity Agent** on top of the style-tracking engine.

The rule of this project:

> We do not predict first. We track how teams play first, then use that tracked identity to explain matchups.

## What this phase does

1. Reads a team-match style log.
2. Creates style ratings from tracked behavior.
3. Builds a Team Identity Agent report.
4. Builds a Matchup Intelligence Agent report.
5. Keeps guardrails so the agent cannot pretend unsupported opinions are facts.

## Why this is not just an Excel model

The project is now structured like a real engine:

- `data/` contains tracked match inputs and scouting notes.
- `src/` contains deterministic code.
- `outputs/` contains reports generated by the engine.
- `tests/` contains sanity checks.
- `docs/` contains design rules and a Codex prompt.

Excel can still be used later as a viewing/reporting layer, but the main engine should live in code.

## Current inputs

The sample match log includes these style-tracking fields:

- possession_pct
- field_tilt_pct
- avg_possession_seconds
- passes_per_possession
- central_progression_pct
- direct_speed_mps
- fast_attacks_per90
- progressive_passes_per90
- progressive_carries_per90
- runs_in_behind_per90
- avg_block_height
- ppda
- high_regains_per90
- opponent_box_touches_allowed
- xga_per90
- avg_team_width
- avg_team_depth
- touch_x_mean
- touch_y_spread
- sprints_per90

These are not meant to be final forever. They are the first measurable version of "how teams play."

## How to run

From this folder:

```bash
python src/run_phase3.py
```

Optional tests:

```bash
pytest -q
```

## Output files

Generated files land in `outputs/`:

- `team_style_summary.csv`
- `team_identity_agent_report.csv`
- `team_identity_agent_report.md`
- `matchup_agent_report.csv`
- `matchup_agent_report.md`

## Important guardrail

This phase does **not** make betting picks. That is intentional.

The agent is only allowed to answer:

- How does this team play?
- What does the tracked data support?
- What style matchup is created?
- Where could the model be wrong?

Prediction comes after we have real historical data and backtesting.
