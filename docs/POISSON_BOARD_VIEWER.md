# Poisson Board Viewer

Phase 26 makes projection checkpoint Poisson outputs readable in the existing static viewer.

The viewer still reads generated files only. It does not recompute projections, download data, enable proxy adjustments, or add current StatsBomb.

## Build

Run a checkpoint with Poisson output and viewer generation:

```powershell
.\.venv\Scripts\python.exe -m src.cli projection-results-checkpoint --as-of-date 2026-06-24 --run-current-international --no-network --allow-sample-data --build-viewer --build-poisson-board
```

Or rebuild the viewer from existing outputs:

```powershell
.\.venv\Scripts\python.exe -m src.cli build-report-viewer --runs-root outputs --output-dir outputs/viewer
```

Open:

```text
outputs/viewer/index.html
```

Checkpoint board pages are written under:

```text
outputs/viewer/projection_checkpoints/YYYY-MM-DD/index.html
```

## What It Shows

Each match card shows:

- projected home xG
- projected away xG
- projected total
- most likely exact score
- top 5 correct-score cells
- 1X2 probability output
- model-implied American odds
- over/under probabilities for 0.5 through 4.5
- BTTS probabilities
- clean sheet and concedes probabilities
- top correct scores
- correct score grid
- data support, confidence, source tier, rating status, and style-input state

Score labels use Excel-safe spacing such as `1 - 0`, not `1-0`.

## Model-Implied American Odds

American odds are a compact way to display a probability-derived price. In this viewer they are calculated directly from model probabilities and are shown only as model-implied context.

They are not betting advice, selections, or wagering guidance.

Exact score cells are naturally low-probability outcomes. The viewer labels the single highest-probability score cell as `Most likely exact score` and shows the top 5 cells so the distribution is easier to inspect.

## Data Support

Data support labels explain how much evidence is behind the projection row:

- real rows came from a source-backed projection input
- manual rows were supplied by the user and are not source-verified
- sample rows are committed demo/test fixtures and are not real current matchups

The board surfaces sample and manual warning banners at the top of the page.

## Style Inputs

Style inputs can be shown as unavailable. That is intentional when the row is rating-only or fixture-only.

The board must not pretend that ratings, fixture lists, or proxy-free baselines are true event/tracking style. Future work can improve the projected home and away xG inputs with measured style-aware adjustments, but this viewer only renders the probability outputs already generated by the checkpoint.
